
# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 2.x.x   | ✅ Yes |
| 1.x.x   | ❌ No  |

## Reporting a Vulnerability

**Please do NOT report security vulnerabilities through public GitHub issues.**

### How to Report
1. **Email:** Contact the core team via [portaldot.io](https://www.portaldot.io)
2. **Discord:** DM a core team member directly
3. **Include:** Description, steps to reproduce, potential impact, suggested fix

### What to Expect
- Acknowledgement within 48 hours
- Status update within 7 days
- Credit in release notes if you wish

## Security Best Practices for Node Operators
- Always run the latest version of portaldot_dev
- Never expose RPC ports (9933, 9944) to public internet without authentication
- Use --rpc-methods Safe in production
- Keep validator keys offline and secure
- Monitor node logs regularly for unusual activity
