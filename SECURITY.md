# Security Policy

## Reporting a Vulnerability

Do not open public GitHub issues for security-sensitive bugs.

Please report suspected vulnerabilities privately to the project maintainers through the contact channels listed on the official Portaldot website or organization profile. Include:

- A clear description of the issue
- Affected components
- Reproduction steps if available
- Any suggested mitigation

Until a dedicated security mailbox is published, avoid disclosing exploit details publicly before the maintainers have had a reasonable chance to investigate and respond.
